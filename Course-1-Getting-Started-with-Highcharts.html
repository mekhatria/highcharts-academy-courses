Lesson 1: Intro to Highcharts

What is Highcharts?

Highcharts is a charting library that makes it easy for developers to create interactive data visualizations in web projects.

  • JavaScript-based: Highcharts is built in JavaScript and works smoothly in web environments without extra dependencies.
  • Client-side: Highcharts runs entirely in the browser, making it portable and easy to integrate into existing projects. This also means Highcharts is backend-agnostic, so it can fetch and visualize data from any server setup or data source.
  • Lightweight and modular: With a small footprint and minimal dependencies, Highcharts remains efficient. Additionally, its modular design lets you include only what you need.
  • SVG-based graphics Highcharts uses SVG (Scalable Vector Graphics). This means chart elements are real DOM nodes that can be styled and animated with CSS or JavaScript, scaled without losing quality, and accessed by other web technologies. Compared to canvas-based rendering, this provides clearer structure, better rendering, and integration with the DOM.

Key Features

Highcharts is flexible as well as easy to use. Beginners can quickly get a chart up and running, while advanced users can dive deeper. Here are some of the key features that make this possible:

  • Responsiveness: In Highcharts, charts automatically adapt to the size of chart container, and they reflow when the browser window is resized to keep the content clear and readable. Additionally, you can add your own responsiveness rules, which can be used to modify the chart design and layout with varying screen sizes.
  • Interactivity: Charts are interactive by default. For example, hovering over a data point shows its value in a tooltip, and the corresponding series highlights dynamically to draw attention to that part of the chart.
  • Smart defaults: Highcharts provides sensible defaults for properties like colors, labels, layout, and more, so even a minimal configuration produces a clean and professional-looking chart.
  • Flexible styling: Charts can be styled both through JavaScript options and via CSS in styled mode, giving developers freedom to choose the approach that best fits their workflow.
  • Data handling: Highcharts supports dynamic and asynchronous updates, making it possible to stream data or refresh charts in real time without reloading the page. It also handles large datasets efficiently with features like boosting.
  • Advanced customization capabilities: For advanced users, Highcharts can be extended and customized far beyond the built-in options. This includes the ability to add new features, draw custom elements, or even change how core parts of the library work. These capabilities make it possible to create unique visualizations and tailor charts to specific project needs.

Key Accessibility Features

Highcharts includes extensive accessibility features. These are important in order to make charts accessible to people with disabilities. Below are some of the key features:

  • Screen reader support: Because of its SVG foundation, chart content is available to screen readers and assistive technologies, so users can get the same information as those viewing the chart visually. Highcharts also includes proper ARIA markup to describe chart elements in ways that assistive technologies understand.
  • Keyboard navigation: Charts can be navigated and interacted with using the keyboard, ensuring users who cannot use a mouse can still explore the data.
  • Accessibility-specific chart options: Developers can configure description text, series labels, or even custom keyboard navigation to make charts more accessible in different contexts.
  • Low vision features: Highcharts supports color schemes such as high contrast mode and also offers pattern fills to improve readability for users with low vision or color blindness.
  • Sonification (audible charts): Charts can be turned into sound using characteristics such as pitch, volume, or timbre to represent data variables. This allows users to interpret data through sound rather than visuals, which is especially beneficial for users with visual impairments.

Additional Advantages

  • Broad browser support: Highcharts is tested across many browsers in their latest version.
  • Mobile friendly: Charts adapt well to mobile devices with touch support and high-DPI rendering, so they remain clear and usable on phones and tablets.
  • Export options: Charts can be exported to PNG, JPEG, PDF, and SVG. In addition, data can be exported to CSV, XLS, and HTML tables. Exporting can be done directly in the client browser, or by using an export server, which can also be self-hosted for full control.
  • Official framework integrations: Highcharts offers official integrations with frameworks like React, Vue, and Angular, making it easier to embed charts in modern web apps.
  • Strong documentation and active community: Highcharts has an intuitive API, is backed by extensive documentation, and has an engaged community, helping developers learn, troubleshoot, and share solutions.

Altogether, the Highcharts suite offers developers a professional, flexible, and accessible way to bring data to life on the web.

The Highcharts Product Suite

In this course, we will be working with Highcharts Core, the main Highcharts library for creating a wide range of charts. But the Highcharts suite consists of several other libraries, each tailored for specific needs. Here is an overview of them and what makes each one unique:

  • Highcharts Core: The core charting library, supporting line, bar, column, pie, and many more chart types, along with many features including those covered in the previous slides.
  • Highcharts Stock: Extends Highcharts Core with additional features, especially relating to timeline navigation and finance.
  • Highcharts Maps: Enables geospatial visualizations, choropleths, and interactive maps, along with several map-specific features.
  • Highcharts Gantt: For project planning and scheduling with tasks, dependencies, and timelines.
  • Highcharts Dashboards: Lets you combine charts, KPIs, tables, and other components into unified, interactive dashboards with built-in data synchronization.
  • Highcharts Grid: A spreadsheet-like data table, available as Grid Lite, which is a free tool that includes basic grid-related features, and Grid Pro, which allow for end user editing and has more advanced features.

Lesson 2: Good to Know Before You Start

System Requirements

Highcharts is based on native browser technologies. To ensure that your setup can run Highcharts smoothly, please make sure your browser is compatible as per the requirements: https://www.highcharts.com/docs/getting-started/system-requirements.

Optional Dependencies

Although Highcharts is self-contained by default, there are some external third party libraries required to enable certain features. These features are linked to client-side exporting and support for latitude/longitude in Highcharts Maps. For more details, see https://www.highcharts.com/docs/getting-started/optional-dependencies.

Source Available

Highcharts is source available software, which means you can view and test the source code (like we are doing in this course), but it is not the same as open source software. The key distinction is that open source software is generally free to use, even for commercial purposes, while source available software like Highcharts comes with specific restrictions, such as requiring a paid license for commercial use. In the case of Highcharts, there are paid commercial licenses, but also free licenses for personal use and educational purposes. Read more at https://www.highcharts.com/blog/education/ and https://shop.highcharts.com/

Course Wrap-Up

Congratulations on completing the Getting Started wtih Highcharts course.

We have covered some key aspects about what Highcharts is, learned how to get a chart up and running, and how to customize it to suit a simple use case.

During this course you learned how to:

  • Create a basic chart using Highcharts
  • Add multiple series and categories to represent your data.
  • Use chart options such as plotOptions, stacking, colors, and dataLabels to change presentation and behavior.
  • Include and use the annotations module to add labels to specific points.
  • Customize how the tooltip is formatted and use event handlers to create additional interactivity.

These fundamentals give you a good starting point for building interactive, production-ready charts. From here you can explore more topics like responsive rules, accessibility options, and integrations with frameworks (React/Vue/Angular).

Note about the accessibility module: In this course, we did not use the accessibility module when creating our chart, as this was outside the scope of the tutorial videos. However, in a production setup, it is highly recommended to include and enable this module to ensure your charts are accessible to all users. To do this, simply add the accessibility module script tag to your HTML, after the Highcharts JS files. Unless you have a specific reason not to, make sure to include it in your project.
